Problem Description

The exportToPNG() function in the code screenshot tool produces completely blank/white images when used on Safari 17.4. The function works correctly on Chrome 122+, Firefox 124+, and Safari 17.3.

The issue was first reported by 3 users on March 3, 2026. Affects approximately 18% of total users (Safari desktop share).

  • Agent investigated font loading timing -- fonts load correctly, issue persists with system fonts only

Relevant Code

src/lib/export.ts 
async function exportToPNG(element: HTMLElement): Promise<Blob> {
  const canvas = await html2canvas(element, {
    scale: 2,
    backgroundColor: null,
    useCORS: true,
    logging: false,
  });

  return new Promise((resolve, reject) => {
    canvas.toBlob(
      (blob) => {
        if (blob) resolve(blob);
        else reject(new Error('Failed to create blob'));
      },
      'image/png',
      1.0
    );
  });
}
src/components/CodePreview.tsx (relevant section) 
const handleExport = async () => {
  if (!previewRef.current) return;
  setExporting(true);
  try {
    const blob = await exportToPNG(previewRef.current);
    const url = URL.createObjectURL(blob);
    downloadFile(url, `codesnap-${Date.now()}.png`);
  } catch (err) {
    toast.error('Export failed. Please try again.');
  } finally {
    setExporting(false);
  }
};
